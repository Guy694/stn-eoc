
คุณคือ Senior Full-stack Developer, UX/UI Designer, GIS Dashboard Architect และผู้เชี่ยวชาญด้านระบบศูนย์ปฏิบัติการฉุกเฉิน EOC

จงออกแบบและสร้างหน้าเมนูใหม่ชื่อ
“สภาพอากาศ / Weather Watch”
สำหรับระบบ
“ระบบศูนย์ปฏิบัติการฉุกเฉิน จังหวัดสตูล”
หรือ
“Satun Provincial Emergency Operations Centers (Satun Geo-EOC)”

เมนูนี้ใช้สำหรับแสดงข้อมูลสภาพอากาศปัจจุบัน พยากรณ์อากาศ ฝนสะสม ความเสี่ยงฝนตกหนัก คลื่นลมแรง และข้อมูลเฝ้าระวังที่เกี่ยวข้องกับน้ำท่วม วาตภัย ดินถล่ม และภัยสุขภาพที่อาจเกิดจากสภาพอากาศ

ให้สร้างหน้า Web Dashboard แบบ Map-first โดยมีแผนที่เป็นศูนย์กลาง และต้องออกแบบให้สอดคล้องกับหน้าระบบ EOC เดิม ได้แก่ Header สีกรมท่า Sidebar เมนูซ้าย KPI Cards แผนที่กลาง Right Panel และ Timeline รายวัน

## 1. เป้าหมายของหน้า Weather Watch

หน้าสภาพอากาศต้องช่วยให้ประชาชนและเจ้าหน้าที่ตอบคำถามเหล่านี้ได้ทันที

1. ตอนนี้จังหวัดสตูลมีสภาพอากาศเป็นอย่างไร
2. พื้นที่ใดมีฝนตกหนักหรือเสี่ยงน้ำท่วม
3. อำเภอใดมีความเสี่ยงสูงใน 24 ชั่วโมงข้างหน้า
4. มีประกาศเตือนภัยฝนหนัก คลื่นลมแรง หรือพายุหรือไม่
5. ประชาชนควรปฏิบัติตัวอย่างไร
6. สามารถดูข้อมูลย้อนหลังหรือพยากรณ์รายวันได้หรือไม่

## 2. Technology Stack

ให้ใช้โครงสร้างเดียวกับระบบเดิม

* Frontend: Next.js หรือ React
* Styling: Tailwind CSS
* UI Components: Shadcn UI หรือ Component ที่ออกแบบให้ทันสมัย

ห้ามฝัง API Key ใน Frontend โดยตรง
ให้เตรียมโครงสร้างให้เรียกผ่าน Backend API Route หรือ Server-side Proxy เท่านั้น

## 14. Risk Scoring Logic

ให้สร้างฟังก์ชันคำนวณระดับความเสี่ยงเบื้องต้น

สูตร:

Weather Risk Score = Rainfall 24h Score + Forecast Rain Score + Wind Score + Flood Susceptibility Score + Warning Score

ตัวอย่างเกณฑ์ Rainfall 24h

* 0–30 มม. = ปกติ
* 31–60 มม. = เฝ้าระวัง
* 61–90 มม. = เสี่ยงสูง
* มากกว่า 90 มม. = วิกฤต

ถ้ามีประกาศเตือนภัยจากหน่วยงานทางการ ให้ยกระดับความเสี่ยงอย่างน้อย 1 ระดับ

ให้แสดง Risk Level เป็น

* normal = ปกติ
* watch = เฝ้าระวัง
* high = เสี่ยงสูง
* critical = วิกฤต

## 16. Data Sources สำหรับอนาคต

ให้เตรียมช่องทางเชื่อมต่อข้อมูลจริงจากแหล่งต่อไปนี้

1. กรมอุตุนิยมวิทยา TMD
   สำหรับพยากรณ์อากาศ ประกาศเตือนภัย ฝน เรดาร์ และข้อมูลอุตุนิยมวิทยา

2. GISTDA
   สำหรับข้อมูลดาวเทียม พื้นที่น้ำท่วม พื้นที่เสี่ยงภัย และข้อมูลภูมิสารสนเทศ

3. OpenWeather API
   สำหรับ current weather, forecast และ weather map layer

4. Windy API
   สำหรับ visual layer เช่น ลม ฝน เมฆ คลื่นทะเล และเรดาร์

5. ข้อมูลภาคสนามจังหวัด
   เช่น รายงานจากอำเภอ อปท. รพ.สต. ปภ. และเจ้าหน้าที่ EOC

ให้เขียนโครงสร้าง Code ให้รองรับการเปลี่ยนจาก Mock Data เป็น API จริงในอนาคต

## 19. ข้อกำหนดสำคัญ

* ต้องออกแบบให้สอดคล้องกับ Satun EOC Dashboard เดิม
* ต้องใช้แผนที่เป็นศูนย์กลาง
* ต้องแสดงข้อมูลแบบรายวันและรายชั่วโมง
* ต้องแสดงระดับความเสี่ยงด้วยสีที่เข้าใจง่าย
* ต้องมีประกาศเตือนภัยสภาพอากาศ
* ต้องมีคำแนะนำประชาชน
* ต้องรองรับข้อมูล 7 อำเภอของจังหวัดสตูล
* ต้องแยกข้อมูลสาธารณะกับข้อมูลภายใน
* ต้องพร้อมเชื่อมต่อ API จาก TMD, GISTDA, OpenWeather หรือ Windy
* ห้ามแสดงข้อมูลส่วนบุคคล


## 20. เป้าหมายสุดท้าย

สร้างหน้า
“สภาพอากาศ / Weather Watch”
ให้เป็นศูนย์กลางติดตามสภาพอากาศและความเสี่ยงจากฝนของจังหวัดสตูล เพื่อสนับสนุนการเฝ้าระวังน้ำท่วม วาตภัย ดินถล่ม และภัยสุขภาพที่เกี่ยวข้องกับสภาพอากาศ โดยประชาชนสามารถดูข้อมูลที่เข้าใจง่าย ส่วนเจ้าหน้าที่สามารถใช้ข้อมูลประกอบการตัดสินใจในระบบ EOC ได้อย่างมีประสิทธิภาพ
