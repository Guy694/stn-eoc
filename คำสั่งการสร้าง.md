คุณคือ Senior Full-stack Developer, UX/UI Designer, GIS Dashboard Architect และผู้เชี่ยวชาญด้านระบบศูนย์ปฏิบัติการฉุกเฉิน EOC ระดับจังหวัด

จงออกแบบและสร้าง Web Application Dashboard สำหรับ
“ระบบศูนย์ปฏิบัติการฉุกเฉินจังหวัดสตูล”
หรือ
“Satun Emergency Operation Center Dashboard”

ระบบนี้ใช้สำหรับแสดงสถานการณ์ภัยพิบัติและเหตุฉุกเฉินของจังหวัดสตูล เช่น น้ำท่วม วาตภัย ดินถล่ม ภัยแล้ง โรคระบาด หรือเหตุฉุกเฉินด้านสาธารณสุข โดยมีเป้าหมาย 2 กลุ่มหลัก คือ

1. ประชาชนทั่วไป
   ใช้ดูข้อมูลสาธารณะ เช่น จุดเกิดเหตุ พื้นที่เสี่ยง ศูนย์พักพิง เส้นทางปลอดภัย ประกาศแจ้งเตือน และช่องทางขอความช่วยเหลือ

2. เจ้าหน้าที่ EOC
   ใช้ติดตามสถานการณ์ วิเคราะห์ผลกระทบ ตัดสินใจ สั่งการภารกิจ จัดสรรทรัพยากร และติดตามผลการดำเนินงานของหน่วยงานต่าง ๆ

ให้สร้างระบบในรูปแบบ Web Dashboard ที่ทันสมัย ใช้งานง่าย มีแผนที่เป็นศูนย์กลางของระบบ และรองรับการแสดงผลทั้ง Desktop, Tablet และ Mobile

## 1. Technology Stack ที่ต้องการ

ให้ใช้ Framework ที่ทันสมัยและเหมาะกับการพัฒนา Dashboard

* Frontend: Next.js หรือ React
* Styling: Tailwind CSS
* Component UI: Shadcn UI หรือ Component Design ที่ดู Professional
* Map: Leaflet, Mapbox หรือ OpenStreetMap
* Chart: Recharts หรือ ECharts
* Icon: Lucide React หรือ Font Awesome
* State Management: ใช้ React State หรือ Zustand
* Data: ใช้ Mock Data ก่อน แต่ต้องออกแบบให้สามารถเชื่อม Google Sheet / CSV / API ได้ในอนาคต

หากยังไม่มี Backend ให้จำลองข้อมูลด้วย Mock JSON ภายในระบบก่อน และออกแบบโครงสร้างข้อมูลให้พร้อมขยายเป็นฐานข้อมูลจริงในภายหลัง

## 2. Design Concept

ออกแบบ UI ให้ดูเป็นระบบบัญชาการเหตุการณ์ฉุกเฉินระดับจังหวัด มีความน่าเชื่อถือ ทันสมัย เป็นทางการ และใช้งานได้จริง

Mood & Tone:

* Government Technology Dashboard
* Emergency Command Center
* GIS Situation Awareness
* Modern Public Safety Platform
* Clean, Fast, Clear, Real-time

โทนสีหลัก:

* Navy Blue: สีหลักของระบบบัญชาการ
* Cyan / Sky Blue: ข้อมูลแผนที่และเทคโนโลยี
* Red / Orange: แจ้งเตือนภัยและพื้นที่วิกฤต
* Green: พื้นที่ปลอดภัยหรือดำเนินการเสร็จสิ้น
* White / Light Gray: พื้นหลังสะอาด อ่านง่าย

รูปแบบการออกแบบ:

* Dashboard แบบ Map-first
* Header ชัดเจน
* Sidebar สำหรับควบคุม Layer แผนที่
* KPI Cards ด้านบน
* Panel รายละเอียดด้านขวา
* สีของ Marker ต้องสื่อระดับความรุนแรง
* ใช้ Badge, Status, Progress Bar, Timeline และ Alert Banner

ให้สร้างหน้า Dashboard หลักอย่างน้อย 2 โหมด

### 3.1 Public Dashboard สำหรับประชาชน

เป็นหน้าที่ประชาชนทั่วไปสามารถเข้าดูได้โดยไม่ต้อง Login

3. Alert Banner

     “แจ้งเตือนน้ำท่วมฉับพลันในพื้นที่อำเภอละงูและทุ่งหว้า”

   * สถานะ

## 4. Logic การจัดระดับความรุนแรง

ให้ใช้ระบบ Risk Score เพื่อจัดระดับพื้นที่บนแผนที่

สูตร:

Risk Score = Hazard Severity + Exposure + Vulnerability + Response Gap

องค์ประกอบ:

1. Hazard Severity
   ระดับความรุนแรงของภัย เช่น ระดับน้ำ ปริมาณฝน ความเร็วลม หรือความรุนแรงของเหตุการณ์

2. Exposure
   จำนวนประชาชน ครัวเรือน ถนน โรงเรียน หน่วยบริการ หรือโครงสร้างพื้นฐานที่ได้รับผลกระทบ

3. Vulnerability
   จำนวนกลุ่มเปราะบาง เช่น ผู้สูงอายุ ผู้ป่วยติดเตียง เด็กเล็ก หญิงตั้งครรภ์ และผู้พิการ

4. Response Gap
   ช่องว่างในการตอบโต้ เช่น ทรัพยากรไม่เพียงพอ ทีมยังไม่ถึงพื้นที่ หรือคำขอความช่วยเหลือค้างนาน

สีแสดงผลบนแผนที่:

* สีเขียว = ปกติ
* สีเหลือง = เฝ้าระวัง
* สีส้ม = กระทบปานกลาง
* สีแดง = วิกฤต
* สีม่วง = วิกฤตรุนแรงมาก ต้องเร่งอพยพหรือสั่งการระดับจังหวัด

## 5. Mock Data ที่ต้องสร้าง

ให้สร้าง Mock Data ของจังหวัดสตูล โดยครอบคลุมอำเภอหลัก ได้แก่

* เมืองสตูล
* ควนโดน
* ควนกาหลง
* ท่าแพ
* ละงู
* ทุ่งหว้า
* มะนัง

ให้จำลองเหตุการณ์หลักเป็น “อุทกภัยจังหวัดสตูล” และสร้างข้อมูลตัวอย่างอย่างน้อย:

* จุดเกิดเหตุ 15 จุด
* พื้นที่ได้รับผลกระทบ 7 อำเภอ
* ศูนย์พักพิง 10 แห่ง
* หน่วยบริการสุขภาพ 20 แห่ง
* ถนนปิด 8 จุด
* คำขอความช่วยเหลือ 30 รายการ
* ภารกิจเจ้าหน้าที่ 20 ภารกิจ
* ทรัพยากร 15 รายการ
* กลุ่มเปราะบางในพื้นที่เสี่ยง

ข้อมูลตัวอย่างต้องมีพิกัด latitude และ longitude ที่อยู่ในพื้นที่จังหวัดสตูลโดยประมาณ เพื่อให้แสดง Marker บนแผนที่ได้จริง

## 7. UX/UI Requirements

ระบบต้องใช้งานง่ายและเหมาะกับภาวะฉุกเฉิน

ข้อกำหนดด้าน UX:

* ผู้ใช้ต้องเห็นสถานการณ์สำคัญภายใน 5 วินาทีแรก
* สีต้องสื่อความหมายชัดเจน
* ตัวเลขสำคัญต้องเห็นชัด
* แผนที่ต้องเป็นศูนย์กลาง
* สามารถกรองข้อมูลตามอำเภอ ตำบล ประเภทภัย และระดับความรุนแรง
* คลิกจุดบนแผนที่แล้วเห็นรายละเอียดทันที
* ต้องมีปุ่ม “ประกาศล่าสุด”
* ต้องมีปุ่ม “ขอความช่วยเหลือ”
* ต้องมีปุ่ม “เปิด/ปิด Layer”
* ต้องมี Responsive Design
* ต้องมี Dark Mode สำหรับห้องบัญชาการ
* ข้อมูลทุกจุดต้องแสดงเวลาอัปเดตล่าสุด

## 10. สิทธิ์การเข้าถึง

ให้ออกแบบ Role-Based Access Control ไว้ในระบบ แม้ระยะแรกจะยังใช้ Mock User

Role ที่ต้องมี:

1. Public User
   เห็นเฉพาะข้อมูลสาธารณะ

2. Field Officer
   เพิ่มรายงานภาคสนาม อัปเดตสถานะภารกิจ และอัปโหลดหลักฐาน

3. EOC Officer
   เห็นข้อมูลสถานการณ์ทั้งหมด จัดการภารกิจ และดูทรัพยากร

4. Commander
   เห็น Decision Support, อนุมัติประกาศ และสั่งการระดับจังหวัด

5. Admin
   จัดการผู้ใช้ ข้อมูลพื้นฐาน และโครงสร้างระบบ

## 11. Public Safety & Privacy

ระบบต้องแยกข้อมูล Public และ Internal อย่างชัดเจน

ข้อมูลที่สามารถเผยแพร่ได้:

* ประเภทเหตุการณ์
* พื้นที่ได้รับผลกระทบในระดับอำเภอ/ตำบล
* ระดับความรุนแรง
* ศูนย์พักพิง
* เส้นทางปลอดภัย
* ประกาศแจ้งเตือน
* คำแนะนำประชาชน
* เบอร์โทรฉุกเฉิน
* เวลาอัปเดตล่าสุด

ข้อมูลที่ห้ามเผยแพร่:

* รายชื่อผู้ประสบภัย
* เบอร์โทรส่วนบุคคล
* พิกัดบ้านรายบุคคล
* ข้อมูลผู้ป่วย
* ข้อมูลกลุ่มเปราะบางแบบระบุตัวตน
* รายละเอียดภารกิจภายใน
* ข้อมูลทรัพยากรที่มีผลต่อความมั่นคง

ข้อความ Header:

“แผนที่สถานการณ์ภัยพิบัติและเหตุฉุกเฉิน จังหวัดสตูล”

ข้อความ Alert:

“แจ้งเตือนเฝ้าระวังน้ำท่วมฉับพลันและน้ำป่าไหลหลากในพื้นที่อำเภอละงู ทุ่งหว้า และมะนัง ขอให้ประชาชนติดตามประกาศจากหน่วยงานราชการอย่างใกล้ชิด”

* ติดตามประกาศจากทางราชการ

* ผู้ป่วยติดเตียง ผู้สูงอายุ และหญิงตั้งครรภ์ ควรแจ้งผู้นำชุมชนหรือเจ้าหน้าที่ใกล้บ้าน

* 1669 แพทย์ฉุกเฉิน

* สำนักงานสาธารณสุขจังหวัดสตูล
* ศูนย์บัญชาการเหตุการณ์จังหวัดสตูล
