# บทคัดย่อ

## 1. บทนำและวัตถุประสงค์

สถานการณ์ภัยพิบัติและเหตุการณ์ฉุกเฉิน เช่น น้ำท่วม ภัยแล้ง แผ่นดินไหว คลื่นยักษ์ และโรคระบาด จำเป็นต้องอาศัยระบบสารสนเทศที่สามารถรวบรวมข้อมูล วิเคราะห์สถานการณ์ และสนับสนุนการตัดสินใจของหน่วยงานที่เกี่ยวข้องได้อย่างรวดเร็วและเป็นระบบ โดยเฉพาะในระดับจังหวัดซึ่งต้องประสานข้อมูลจากหลายพื้นที่ หลายหน่วยงาน และหลายบทบาทการปฏิบัติงาน ระบบศูนย์บัญชาการเหตุการณ์ฉุกเฉินจังหวัดสตูล หรือ STN EOC จึงถูกพัฒนาขึ้นเพื่อเพิ่มประสิทธิภาพการบริหารจัดการเหตุการณ์ฉุกเฉิน การติดตามสถานการณ์ผ่านแผนที่ การจัดการรายงาน การกำหนดทีมปฏิบัติงาน และการควบคุมสิทธิ์การใช้งานตามบทบาท

การศึกษานี้มีวัตถุประสงค์เพื่อพัฒนาและประเมินแนวทางการออกแบบระบบสารสนเทศสำหรับศูนย์บัญชาการเหตุการณ์ฉุกเฉินที่มีความยืดหยุ่น รองรับภัยหลายประเภท สามารถจัดการข้อมูลเชิงพื้นที่และรายงานสถานการณ์ได้อย่างเป็นระบบ รวมทั้งสนับสนุนการยืนยันตัวตน การมอบหมายทีมงาน และการเข้าถึงข้อมูลตามบทบาทของผู้ใช้งาน

## 2. วิธีการศึกษา

การศึกษานี้เป็นการวิจัยเชิงพัฒนา โดยใช้แนวทางการวิเคราะห์ ออกแบบ พัฒนา และทดสอบระบบสารสนเทศ กลุ่มตัวอย่างและขอบเขตการใช้งานกำหนดจากผู้มีส่วนเกี่ยวข้องกับการบริหารจัดการเหตุการณ์ฉุกเฉิน ได้แก่ ผู้ดูแลระบบ ผู้จัดการ EOC เจ้าหน้าที่ EOC เจ้าหน้าที่ท้องถิ่น และผู้ใช้งานทั่วไป การคัดเลือกกลุ่มตัวอย่างใช้วิธีเลือกแบบเจาะจงตามบทบาทหน้าที่และความเกี่ยวข้องกับกระบวนการปฏิบัติงานของศูนย์บัญชาการเหตุการณ์

เครื่องมือที่ใช้ในการพัฒนาระบบประกอบด้วย Next.js และ React สำหรับส่วนติดต่อผู้ใช้ Next.js API Routes สำหรับส่วนบริการข้อมูล MySQL สำหรับฐานข้อมูล Leaflet สำหรับการแสดงแผนที่ และระบบยืนยันตัวตนที่รองรับทั้ง username/password และ ThaiID OAuth 2.0 เครื่องมือเก็บรวบรวมข้อมูลประกอบด้วยการวิเคราะห์ความต้องการของระบบ การตรวจสอบโครงสร้างฐานข้อมูล การทดสอบฟังก์ชันการทำงาน และการบันทึกกิจกรรมการใช้งานของผู้ใช้ การวิเคราะห์ข้อมูลใช้การวิเคราะห์เชิงพรรณนา โดยพิจารณาความครบถ้วนของฟังก์ชัน ความสอดคล้องกับกระบวนการทำงาน ความถูกต้องของข้อมูล และความสามารถในการขยายระบบเพื่อรองรับภัยหลายประเภท

## 3. ผลการศึกษา

ผลการศึกษาพบว่าระบบ STN EOC สามารถรองรับการบริหารจัดการเหตุการณ์ฉุกเฉินในระดับจังหวัดได้หลายมิติ ได้แก่ การจัดการประเภทภัยพิบัติ การแสดงผลข้อมูลบนแผนที่เชิงพื้นที่ การติดตามข้อมูลระดับหมู่บ้าน การจัดทำรายงานสถานการณ์ การบริหารทีมปฏิบัติงานในแต่ละ session ของ EOC และการควบคุมสิทธิ์การใช้งานตามบทบาท ระบบมีโครงสร้างแบบยืดหยุ่นโดยสามารถกำหนดโมดูลหรือเมนูที่แตกต่างกันตามประเภทภัย เช่น น้ำท่วม ภัยแล้ง และโรคระบาด ทำให้การใช้งานสอดคล้องกับลักษณะข้อมูลและภารกิจของแต่ละเหตุการณ์

นอกจากนี้ ระบบยังรองรับการจัดการทีม EOC แบบ dynamic โดยเจ้าหน้าที่สามารถมีบทบาทต่างกันในแต่ละเหตุการณ์ มีการบันทึกข้อมูลทีม สมาชิกทีม และสิทธิ์การเข้าถึงโมดูลของระบบอย่างเป็นระบบ ส่วนการยืนยันตัวตนรองรับทั้งการเข้าสู่ระบบแบบปกติและการยืนยันตัวตนผ่าน ThaiID ช่วยเพิ่มความน่าเชื่อถือของผู้ใช้งานและลดความเสี่ยงด้านความปลอดภัย ขณะเดียวกัน dashboard และระบบรายงานช่วยให้ผู้ปฏิบัติงานสามารถติดตามสถานการณ์และข้อมูลสำคัญได้รวดเร็วขึ้น

## 4. อภิปรายผลและข้อเสนอแนะ

ระบบ STN EOC แสดงให้เห็นว่าการออกแบบระบบสารสนเทศสำหรับศูนย์บัญชาการเหตุการณ์ควรเน้นความยืดหยุ่นของโครงสร้างข้อมูล การแบ่งสิทธิ์ตามบทบาท และการแสดงผลข้อมูลที่สนับสนุนการตัดสินใจเชิงพื้นที่ การแยกโมดูลตามประเภทภัยช่วยให้ระบบสามารถรองรับลักษณะงานที่แตกต่างกันได้โดยไม่จำเป็นต้องปรับโครงสร้างหลักใหม่ ขณะที่การจัดการทีมแบบ session-based ช่วยสะท้อนการปฏิบัติงานจริงซึ่งเจ้าหน้าที่อาจเปลี่ยนบทบาทได้ตามเหตุการณ์

ข้อเสนอแนะคือ ควรทดสอบระบบกับผู้ใช้งานจริงในสถานการณ์จำลองหรือสถานการณ์ปฏิบัติงานจริงเพื่อประเมินความสะดวก ความถูกต้อง และความรวดเร็วในการใช้งาน ควรพัฒนาระบบแจ้งเตือนและการเชื่อมต่อข้อมูลจากหน่วยงานภายนอกเพิ่มเติม เช่น ข้อมูลสภาพอากาศ ข้อมูลดาวเทียม หรือข้อมูลสาธารณสุข รวมทั้งควรเพิ่มการประเมินด้านความปลอดภัยของข้อมูลและการสำรองข้อมูล เพื่อให้ระบบสามารถใช้งานได้อย่างต่อเนื่องและน่าเชื่อถือในภาวะฉุกเฉิน

## 5. คำสำคัญ

ศูนย์บัญชาการเหตุการณ์ฉุกเฉิน, ระบบสารสนเทศ, ภัยพิบัติ, แผนที่เชิงพื้นที่, การจัดการเหตุการณ์, ThaiID, การควบคุมสิทธิ์ตามบทบาท
